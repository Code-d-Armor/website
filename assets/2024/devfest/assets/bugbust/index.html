DevFest BugBust Challenge

Ce que nous proposons :

AWS propose un nouveau service permettant d’organiser des concours de correction de code : AWS BugBust !

Ce service est fait pour corriger les bugs de vos propres bases de code mais ici nous vous proposons de tester ce service en participant à un évènement créé pour l’occasion et intégrant des repos publics choisis par l’équipe du DevFest.

(Le secret (pas si secret) d’AWS BugBust est que la détection des bugs à fixer sont identifiés par un algorithme de machine learning et permet d’automatiser les revues de code grâce à AWS CodeGuru Reviewer!)

Ici, nous vous proposons de participer au premier BugBust en France et, peut-être, gagner des cadeaux en fixant le plus de bugs cette semaine !

Prérequis :

  • Un compte Github

Comment accéder au challenge AWS Bugbust du Dev Fest :

Etape 1

Il vous faut accéder au site suivant : https://devfest-bugbust.awsapps.com/start/

Ensuite, cliquer sur le bouton pour vous authentifier avec votre compte github

Etape 2

Utiliser votre login / mot de passe github.

Etape 3

Autoriser l’application DevFestBugBust dans Github

Etape 4

Cliquer sur AWS Account, qui vous permettra d’étendre le menu

Etape 5

Vous pouvez maintenant cliquer sur Management Console

Etape 6

Cela vous permet d’accéder à la page BugBust du DevFest.

Vous allez maintenant rejoindre l’évenement en cliquant sur le bouton “join event”

Etape 7

Vous allez maintenant créer un compte dans AWS BugBust, en cliquant sur “Create an AWS BugBust Player account”

Etape 8

S’assurer de bien avoir cliqué sur “Sign up”.

Etape 9

Créer un compte AWS BugBust avec le même email que Github (c’est important car ce sera le moyen de lier vos “pull requests” à votre compte BugBust).

Etape 10

Un code va vous être envoyé pour validation de votre email

Etape 11

Rappel qu’il faut que l’email autorisé dans BugBust soit celui de votre compte Github (Si celui utilisé pour l’inscription est différent, vous pouvez rajouter votre email github dans les paramètres de votre compte BugBust)

Etape 12

Vous pouvez maintenant vous lancer dans le challenge

Comment résoudre un bug et gagner des points :

Etape 1: “Claim a bug”

Vous pouvez parcourir la liste des bugs à résoudre

et choisir de travailler sur jusqu’à 10 bugs en parallèle en cliquant sur “Claim”

Etape 2: Mettre en place votre environement de développement

Dans un premier temps, il vous faudra identifier quelle base de code est sujet au bug choisi: “sdc” dans l’example suivant

rendez vous sur le github du devfest : https://github.com/devfest-bugbust/ et en particulier sur le “repository” identifié précédemment

forkez ensuite ce repo dans votre compte en cliquant sur fork

Enfin vous pouvez soit cloner le repo sur votre machine local.
Pensez à bien configurer votre email princiaple de github avec la commande : git config user.email "<email>"

Vous pouvez aussi utiliser la nouvelle fonctionnalité de github (github codespace) en remplaçant simplement le “.com” en “.dev” dans votre barre de navigation pour obtenir un IDE en ligne.

Vous pouvez maintenant commencer à travailler sur le code et identifier le bug à résoudre en vous reportant au dashboard de bugbust et en naviguant dans votre environnement de développement.

(ligne 39 de “MigrationMenu.java” dans l’exemple)

Etape 3: Soumettre votre “Pull Request”

Après avoir corrigé le bug choisi

Créez un nouveau “commit” sur votre “fork” (flochaz/sdc master dans l’exemple)

Puis soumettez une “Pull request” sur le “repository” d’origine (devfest-bugbust/sdc)

Après quelques minutes, vous retrouverez votre “pull request” dans le tableau de AWS BugBust avec un status “Analyzing”

Quand celle-ci aura été analysée par AWS CodeGuru Reviewer, elle apparaîtra avec un status “Waiting for merge” si AWS CodeGuru accepte votre proposition de résolution (c’est à dire qu’il ne trouve plus de recommandation à faire sur cette partie du code). Cette étape peut prendre plusieurs minutes.

Enfin, quand votre “Pull Request” aura été mergé par l’organisateur, vos points vous seront délivrés

Vous pourrez voir votre classement dans sur le Leaderboard !!!
